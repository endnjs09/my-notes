# 1. 합성(가짜) 데이터 생성
모델이 학습을 하기 위해서는 데이터가 필요하다. 실제 데이터셋을 사용할 수도 있지만 여기선 선형 회귀의 원리를 확인하기 위해 직접 설계한 합성(가짜) 데이터를 만들어서 사용할 것이다.<br>

여기서는 총 3단계로 데이터를 설계하는데, <br>

- 합성 데이터를 만들어서 사용하는 것이기 때문에 가중치($w$)와 편향($b$)의 정답을 미리 정해둬야 한다. 이렇게 해야 나중에 모델이 학습한 결과가 진짜 정답에 얼마나 가까워졌는지 정확히 평가할 수 있기 때문이다.

- 3-1에서 말했듯이 현실의 데이터는 완벽한 직선 위에 있지 않다. 모델이 너무 쉬운 방향으로 나아가지 않도록 무작위 노이즈(Noise)를 섞어서 현실감을 줘야한다.

- 마지막으로 전체 데이터를 학습용(Train)과 검증용(Validation)으로 나눈다. 학습용은 모델이 가중치를 수정하면서 원리를 깨우치는 데 사용하는 연습 문제이고, 검증용은 학습에 사용하지 않은 새로운 데이터로 일종의 모의고사 느낌이다.
<br><br>

```python
# 데이터 생성
def create_data(w, b, num_train, num_val):  # 가중치, 편향, 학습용, 검증용
    num_inputs = len(w)              # 특징의 개수
    n = num_train + num_val          # 전체 데이터 개수
    X = torch.randn(n, num_inputs)   # (전체 데이터 개수 x 특징의 개수) 행렬
    y = torch.matmul(X, w.reshape((-1, 1))) + b  
    # X: 입력데이터(데이터 개수, 특징 개수)
    # w.reshape((-1,1)): (200,1) 가로 모양이면 세로로 세워줌
    # b: 편향
    # 즉 위 코드는 y = Xw + b 식을 의미함

    y += torch.randn(y.shape) * 0.01  # 노이즈 추가

    # 훈련셋과 검증셋 분리
    train_ds = TensorDataset(X[:num_train], y[:num_train])   # 슬라이싱
    val_ds = TensorDataset(X[num_train:], y[num_train:])

    return train_ds, val_ds
```
예를 들어 성적을 예측하기 위해 공부 시간($x_1$), 출석 횟수($x_2$), 과제 점수($x_3$)를 본다고 가정해보자. 이때 **num_input** 즉, 특징의 개수는 3이된다. 그리고 이 3가지 정보 각각에 대해 가중치($w$)도 3개가 필요할 것이다. <br>


**TensorDataset**은 torch.utils.data에서 제공하는 데이터 포장지이다. 문제지($X$)와 정답지($y$)는 원래 각각 따로 노는 거대한 숫자 덩어리(Tensor)일 뿐인데, 이를 세트로 묶어주는 역할을 한다. $X$의 0번 데이터와 $y$의 0번 데이터를, $X$의 1번 데이터와 $y$의 1번 데이터를 즉, $n$번은 $n$번끼리 짝을 지어주는 것이다. 이전에는 X[0] 따로, y[0] 따로 불러야 했다면, 이제는 train_ds[0] 한 번만 호출해도 (문제, 정답) 한 쌍이 튜플(Tuple) 형태로 나오게된다.
<br><br><br>

# 2. 모델 설계
앞서 알아본 이론들에 비해 코드는 생각보다 단순하다.

```python
# 모델 설계
class LinearRegression(nn.Module):  # nn.Module
    def __init__(self, num_inputs):
        super().__init__()
        self.net = nn.Linear(num_inputs, 1)  
        # y = Xw + b 연산 (내부적으로 weight(w)와 bias(b)라는 텐서를 자동으로 만듦)

        nn.init.normal_(self.net.weight, mean=0, std=0.01)   
        # 가중치를 평균 0, 표준편차 0.01인 정규분포 값으로 채움

        nn.init.zeros_(self.net.bias)    
        # 편향(b)는 0으로 초기화
        # _(언더바)는 텐서 값을 복사하지 않고 즉시 바꿈

    def forward(self, X):
        return self.net(X)
```
num_inputs 개의 입력을 받아서 1개의 결과($y$)를 내는 선형회로를 만들고, 가중치를 무작위 숫자로 채워 넣고 (편향은 0), 데이터가 들어오면 이 선형회로를 통과시켜 결과를 내뱉게 된다.<br>

참고로 nn.Module은 모든 Pytorch 모델의 조상격이 되는 기본 클래스이다. nn.Module을 상속받는 순간 단순한 클래스(깩체)가 아니라 가중치를 추적할 수 있고, GPU로 이동할 수 있으며 미분이 가능한 지능형 객체가 된다. Pytorch의 기능을 쓰려면 예외없이 거의 필수적이라고 한다.
<br><br><br>

# 3. 학습(훈련)
생성한 데이터와 모델을 바탕으로 실제로 모델이 훈련하는 일련의 과정을 나타낸 코드이다.

```python
# 실제 훈련 루프 
def train_model(model, train_ds, val_ds, lr, wd, epochs, batch_size):
   
    # DataLoader: TensorDataset으로 포장한 데이터를 배치 단위로 잘라서 배달
    train_loader = DataLoader(train_ds, batch_size=batch_size, shuffle=True)
    val_loader = DataLoader(val_ds, batch_size=batch_size)

    # 손실 함수와 최적화 도구
    criterion = nn.MSELoss()     # MSE(평균제곱오차)
    optimizer = torch.optim.SGD(model.parameters(), lr=lr, weight_decay=wd)

    for epoch in range(epochs):
        # 학습 모드
        model.train()
        train_loss = 0

        for X_batch, y_batch in train_loader:
            optimizer.zero_grad()        # 1. 기울기 초기화
            y_hat = model(X_batch)       # 2. 예측
            loss = criterion(y_hat, y_batch) # 3. 손실 계산
            loss.backward()              # 4. 역전파
            optimizer.step()             # 5. 가중치 업데이트
            train_loss += loss.item()

        # (그래프) 에포크 평균, 학습 손실 계산 / 저장
        avg_train_loss = train_loss / len(train_loader)
        train_loss_history.append(avg_train_loss)

        # 검증 모드
        model.eval()
        val_loss = 0
        with torch.no_grad(): # 검증 시에는 기울기 계산을 꺼서 메모리 절약
            for X_val, y_val in val_loader:
                y_hat_val = model(X_val)
                v_l = criterion(y_hat_val, y_val)
                val_loss += v_l.item()
        
        # (그래프) 에포크 평균, 학습 손실 계산 / 저장
        avg_val_loss = val_loss / len(val_loader)
        val_loss_history.append(avg_val_loss)

        print(f"Epoch {epoch+1}/{epochs} | Train Loss: {train_loss/len(train_loader):.6f} | Val Loss: {val_loss/len(val_loader):.6f}")

    return train_loss_history, val_loss_history
```
먼저 학습할 데이터가 필요하므로 '데이터 생성'에서 포장한 데이터(TensorDataser)에서 배치 단위로 잘라서 가져온다. 모델이 달달 외우는 걸 막기 위해 매 에포크(주기)마다 섞어줘야한다(shuffle). <br>

사용할 손실함수와 최적화 도구를 세팅한다. 손실함수로는 평균제곱오차(MSE)를 사용할 것이다. optimizer 안에 weight_decay라고 써있는 걸 볼 수 있는데 이게 3-6에서 알아볼 가중치 감쇠이다. 지금은 이런게 있구나 라고만 보고 넘어가자. <br>

이제 모델이 학습을 하는데 다음과 같은 순서로 진행된다.
- 1. 다음 문제를 풀 때 지장이 없도록 이전 계산에서 나온 기울기(Gradient)를 초기화한다.
- 2. 데이터를 모델에 넣으면, forward 함수가 실행되어 $Xw + b$ 계산을 하고 예측 값을 내뱉는다.
- 3. 내뱉은 예측 값과 실제 정답을 비교해서 오차(Loss)를 계산한다.
- 4. (핵심) '역전파'라고 부르는 과정이 일어난다. 오차를 바탕으로 각 가중치가 얼마나 잘못됐는데 미분값을 계산한다. 우리가 배운 경사하강법이 여기서 쓰인다.
- 5. 계산이 끝나면 가중치($w$)와 편향($b$)을 수정한다.
<br>

위 1번부터 5번까지의 과정을 계속 반복한다. 정확히는 학습 데이터가 1000개 배치 크기가 5라면 한 에포크(epochs)당 200번 수행하게 된다. 에포크를 10으로 설정했다면 총 2000번을 훈련하게 되는 것이다.
<br>

학습이 끝났으면 검증을 하는데 이때는 절대 가중치, 편향을 고치지 않는다. 기울기를 계산하지 않으므로 메모리를 아낄 수 있고 연산 속도도 높아진다.
<br><br><br>


데이터 생성, 모델 설계, 훈련을 바탕으로 이제 직접 실행해보자.
