# 1. 가중치 감쇠(Weight Decay)
3-5에서 언급한 오버피팅(과적합)은 모델을 설계할 때 중요한 문제이다. 모델이 작은 노이즈(Noise)까지 달달 외워버리면 데이터의 아주 작은 노이즈에도 반응하려고 가중치를 수천, 수만 배로 확 키워버린다.<br>

그렇기 때문에 가중치를 강제로 깎아내려서 억지로 작게 유지하는 것이 가중치 감쇠의 핵심이다. 웬만한 노이즈는 무시하고 정말 중요한 특징에만 반응하게 된다.
$$L(\mathbf{w}, b) + \frac{\lambda}{2} \|\mathbf{w}\|^2$$
- $\|\mathbf{w}\|^2$($L_2$ Norm): 가중치들의 제곱 합인데, 가중치가 커지면 이 값도 커진다
- $\lambda$ (람다): 우리가 정하는 벌금(가중치 감쇠 크기)이다. (3-4에서 말한 weight_decay가 바로 이것이다.)

이제 모델은 오차(Loss)만 줄이는 게 아니라, '오차도 줄이면서 가중치 크기도 작게 유지'하는 방향으로 나아가게 되는데, 동시에 가중치를 업데이트 하는 로직도 변하게 된다.<br>
$$\begin{aligned}
\mathbf{w} & \leftarrow \left(1- \eta\lambda \right) \mathbf{w} - \frac{\eta}{|\mathcal{B}|} \sum_{i \in \mathcal{B}} \mathbf{x}^{(i)} \left(\mathbf{w}^\top \mathbf{x}^{(i)} + b - y^{(i)}\right).
\end{aligned}$$
원래는 기울기(Gradient)만큼 뺐는데 이제는 $(1 - \eta\lambda)$ 라는 계수가 앞에 붙는데, 매 단계마다 원래 값에서 일정 비율을 강제로 깎아낸 뒤에 이동하게 되는 것이다. 
<br><br>

훈련용 데이터 수 20개, 검증용 데이터수 100개를 세팅해보자. 훈련용 샘플이 적다보니 모델은 문제의 규칙이나 원리를 찾기 보단 문제 째로 외우려고 할 것이다. 먼저 wd(Weight_Decay) 값을 0으로 했을 때의 결과이다.

```python
num_inputs = 200
true_w = torch.ones((num_inputs, 1)) * 0.01
true_b = 0.05

# 데이터 생성
train_ds, val_ds = create_data(true_w, true_b, num_train=20, num_val=100)
model = VanillaLinearRegression(num_inputs)

epochs = 10
train_hist, val_hist = train_model(model, train_ds, val_ds, lr=0.01, wd=0, epochs=epochs, batch_size=5)
```
