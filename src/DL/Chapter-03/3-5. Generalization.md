# 1. 일반화(Generalization): 암기인지, 이해인지
우리가 시험 공부를 할 때 기출문제를 반복해서 푸는 이유는 실제 시험장에서 처음 보는 유형의 문제에 대비하기 위해서일 것이다. 하지만 기출문제의 문제와 답만 통째로 외웠다면 조금만 숫자가 변형된 문제가 나와도 틀리게 된다. <br>
반대로 통째로 외우는 게 아니라 문제에 담긴 공식과 원리를 제대로 이해했다면 새로운 문제가 나와도 해결할 수 있을 것이다. <br>

일반화가 중요한 이유는 과거의 데이터를 정리하기 위해서가 아니라 과거의 데이터를 갖고 앞으로 일어날 일을 예측하기 위해서이다. 

1) **언더피팅**(Underfitting): 모델이 너무 단순해서 훈련 데이터조차 제대로 못 맞추는 상태. 예를 들어 복잡한 곡선 데이터를 직선 하나로 설명하려 할 때. <br>

2) **오버피팅**(Overfitting): 모델이 너무 복잡해서 데이터의 '노이즈'까지 다 외워버린 상태. 훈련 오차는 0에 가깝지만, 실전(검증) 오차는 치솟는다. <br>

3) **최적**(Optimum): 훈련 오차와 검증 오차가 모두 낮으면서 그 간격이 좁은 지점. 우리가 찾아야 할 '골디락스' 지점
<br>

일반화가 잘 되었다는 것은 모델이 데이터에 숨겨진 본질적인 규칙과 원리를 찾아냈다는 뜻이다. 
<br><br>

### 데이터의 삼분할: 훈련, 검증, 테스트
실제 개발에서는 데이터를 세 주머니로 나뉘는데,

- 훈련 세트(Training set): 모델이 직접 보고 공부하는 문제집.
- 검증 세트(Validation set): 여러 모델 중 "어떤 놈이 제일 똑똑한가?" 혹은 "학습률을 얼마나 줄까?" 결정할 때 쓰는 모의고사. (이걸로 모델 선택을 한다)
- 테스트 세트(Test set): 딱 한 번만 열어보는 최종 수능 시험. 절대 모델 선택 과정에서 이 데이터를 봐서는 안 된다. (자신을 속이는 꼴이 되기 때문)
<br><br>

### K-폴드 교차 검증 (K-fold Cross-Validation)
K-폴드 교차 검증은 데이터가 너무 적어서 검증 세트를 따로 떼놓기 아까울 때 쓰는 전략으로, 데이터를 $K$개(보통 5, 10)의 덩어리로 나눈다.
<br>

1번 덩어리를 검증용으로 쓰고 나머지를 훈련용으로 쓰는데, 이 과정을 2번, 3번... 5번 덩어리가 검증용이 될 때까지 반복한다. 나온 5개의 점수를 평균 내서 최종 실력을 평가한다. 데이터를 한 방울도 낭비하지 않는 알뜰한 방법이다.
