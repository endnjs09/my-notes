# 1. 분류 (Classification)
회귀는 '이 데이터의 값은 얼마인가?'와 같이 연속적인 숫자를 예측하는 거라면 분류는 '이 데이터는 어느 것인가?'로 범주(확률)를 예측하는 것이다. 즉, 회귀에서는 정답(출력)이 하나였지만 분류에서는 출력이 여러 개($o_1, o_2, o_3$)로 늘어난다.<br> 

그렇기 때문에 각 카테고리마다 이게 정답일 가능성(졈수, $o_1, o_2, o_3...o_n$)를 따로 매겨야 한다. 예를 들어 특징은 4개($x_1, x_2, x_3, x_4$)이고, 카테고리는 3개(예: 개, 고양이, 닭)인 상황이라고 가정해보자.<br>
- $o_1$: "이 데이터가 개일 점수는 몇 점이지?"
- $o_2$: "이 데이터가 고양이일 점수는 몇 점이지?"
- $o_3$: "이 데이터가 닭일 점수는 몇 점이지?"
<br>

총 3개의 식이 필요해지는데,

$$\begin{split}\begin{aligned}
o_1 &= x_1 w_{11} + x_2 w_{12} + x_3 w_{13} + x_4 w_{14} + b_1\\
o_2 &= x_1 w_{21} + x_2 w_{22} + x_3 w_{23} + x_4 w_{24} + b_2\\
o_3 &= x_1 w_{31} + x_2 w_{32} + x_3 w_{33} + x_4 w_{34} + b_3
\end{aligned}\end{split}$$

출력 $o_1$ 하나를 계산하기 위해 입력값 4개를 모두 사용하는데 ($o_2, o_3$도 마찬가지), 입력 4개와 출력 3개를 곱하면 가중치($w$)는 총 $4 \times 3 = 12$개가 필요하다.<br>

카테고리가 무수히 많을 경우 위에처럼 식을 다 따로 쓰기엔 한계가 있다. 그래서 아래와 같이 한 줄로 정의한다.
$$\mathbf{o} = \mathbf{W} \mathbf{x} + \mathbf{b}$$
<br><br>

# 2. 소프트맥스(Softmax)
분류에서 선형 모델($\mathbf{o} = \mathbf{W} \mathbf{x} + \mathbf{b}$)을 그냥 돌린 결과를 보면 [10.5, -2.1, 0.3, ...] 같이 숫자가 제각각으로 나오게 된다. <br>
이걸 그대로 쓰면 이 사진이 개, 고양이, 닭인 걸 얼마나 모델이 확신하는지 알기 어렵다. 그래서 **소프트맥스 함수**라는 필터를 씌운다. <br>

$$\hat{y}_i = \frac{\exp(o_i)}{\sum_j \exp(o_j)}$$

모든 숫자를 0 ~ 1 사이로 만들고, 모든 출력의 합을 1(100%)로 만들어서 '이 사진은 90% 확률로 개입니다' 라고 해석 가능한 확률 분포를 얻게 된다. 참고로 지수함수(exp)를 쓰는 이유는 단순히 숫자를 합으로 나누는 게 아니라 큰 숫자를 더 크게 부풀리고 음수를 양수로 바꾸기 위해서이다.
<br><br><br>

# 3. 교차 엔트로피(Cross-Entropy)
교차 엔트로피가 뭔지 알아보기 전에 엔트로피(Entropy)와 놀람의 정도(Surprisal)라는 개념에 대해 먼저 짚고 가야한다.
1) 엔트로피(무질서도): 어떤 시스템(or 확률분포)이 가진 평균적인 놀람의 정도를 말한다. 예를 들어 어떤 주머니가 있는데 이 주머니에서 뭐가 나올지 얼마나 예측하기 힘든가(불확실성)을 숫자로 나타낸 것이다.
    - 낮은 엔트로피: 질서 정연한 상황으로, 주머니에 빨간 공만 100개 들어있다고 해보자. 주머니에서 꺼낸 공은 항상 빨간 공일 것이다. 불확실성이 낮고 엔트로피도 낮다
    - 높은 엔트로피: 무질서인 상황으로, 주머니에서 뭐가 나올지 알 수가 없다. 불확실성이 높고 엔트로피도 높다.
<br>

2) 놀람의 정도: 정보 이론에서 어떤 사건의 정보량은 그 사건이 일어날 확률의 반대를 의미한다. 즉, 일어날 확률이 낮을 수록 놀람의 정도는 커진다. <br>
예를 들어, 해가 동쪽에서 뜬다는 것은 누구나 다 아는 사실이다. 정보 가치가 없고 아무도 놀라지 않을 것이다. 반대로 전교 꼴찌인 친구가 서울대에 합격한 사실을 들었다. 엄청난 빅뉴스이기 때문에 정보 가치가 매우 높고 모두가 놀랄 것이다.
<br>

교차 엔트로피는 이런 이론에서 탄생한 개념이다. 모델의 예측 분포를 정답 분포에 가깝게 만드는 것, 즉 모델이 정답을 보고도 놀라는 정도를 최소화 하는 것이 목표인 것이다. 
<br>

회귀 모델에서는 오차(Loss)를 측정하기 위해 MSE(평균제곱오차)를 사용했다. 분류 모델에서는 오차를 측정할 때 **교차 엔트로피 손실 함수**를 사용한다.<br>
$$l(\mathbf{y}, \hat{\mathbf{y}}) = - \sum_{j=1}^q y_j \log \hat{y}_j$$
모델이 얼마나 틀렸는지를 나타내는 척도로 우리가 줄여야할 최종 목표이다. 모델이 정답 칸에 대해 예측한 확률이 낮을수록 벌점을 기하급수적으로 세게 준다. (로그함수를 사용하는 이유이기도 하다.) <br>

이 손실 함수를 미분하면 어떻게 될까? 이게 여기서의 핵심이다. 손실 함수($l$)를 우리가 만든 점수($o_j$)로 미분(기울기 계산)했을 때 결과는 다음과 같다. (수학 계산하는 법을 공부하는 게 아니니까 미분 과정은 생략)
$$\partial_{o_j} l(\mathbf{y}, \hat{\mathbf{y}}) = \text{softmax}(\mathbf{o})_j - y_j$$
- $\text{softmax}(\mathbf{o})_j$: 모델이 내놓은 예측값 (예: '이건 70% 확률로 고양이이다.')
- $y_j$: 실제 정답 (예: "이건 100% 고양이이다.")
<br>

결과는 (예측 - 실제 정답)라는 오차 신호가 나오는데, 모델은 이걸 보고 가중치를 고친다. 어디서 본 것 같다면 그렇다, 회귀의 손실함수 미분 결과와 완전히 똑같다.
