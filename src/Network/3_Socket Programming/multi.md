# 다중처리 기술
서버는 여러 클라이언트의 요청을 처리해야할 경우가 많음. 클라이언트의 경우 키보드 입력, 모니터 출력, 네트워크 송수신 등을 병행하여 처리해야할 필요가 있음. <br
동시에 여러 작업을 처리해야 할 경우 이를 다중처리라고 부르기도 함. 

## 1. 멀티태스킹
**멀티태스킹**은 여러 작업을 여러 프로세스(or 스레드)들이 나누어 병행하여 처리하는 방법을 말함. 멀티 태스킹을 위해서 프로세스를 여러개 실행 시키거나(멀티프로세스), 스레드를 여러기 실행시키는 방법(멀티 스레드)을 사용

### 1.1 멀티 프로세스
**멀티프로세스**는 독립적으로 처리해야 할 작업의 수만큼 프로세스를 만드는 방법이다. 
- 장점: 멀티프로세스로 다중처리를 하게 되면 각 프로세스들이 독립적으로 작업을 처리하며로 구현이 간편함
- 단점: 
  - 프로세스 수 증가로 인한 프로그램 성능 저하
  - 프로세스간 데이터 공유를 위해 IPC를 사용해야 함
  - IPC를 사용할 경우 프로그램 복잡성이 증가함


### 1.2 멀티 스레드
**스레드**는 프로세스 내에서 독립적으로 실행되는 작업 단위를 말하는데, 프로새스 내에서 여러 스레드를 실행시켰을 때 외부에서는 이 스레드들 전체가 하나의 프로세스처럼 취급됨. 프로세스에 비해 필요한 공간이 적고 생성시간이 짧음
<br>

프로세스에서 스레드를 생성하면 프로세스 이미지를 복사하여 사용하는 것이 아니라, 원래 프로세스의 이미지를 공유하여 같이 사용함. (단, 새로 생성된 스레드용 스택 영역은 별도 배정되기 때문에 스레드들이 스택은 공유하지 않음)
<br>

프로세스 내에서 생성된 스레드들은 이미지를 공유하므로 전역변수를 같이 사용함. 여러 스레드가 한 데이터에 동시에 접근하게 되면 동기화 문제와 임계영역 문제가 발생함.
<br><br><br>


## 2. 다중화 (멀티플렉싱)
한 프로세스(스레드)내에서 여러 작업을 돌아가며 처리하는 것을 말함. <br>

**블로킹(Blocking)**: 수신 데이터가 없으면 데이터가 도착할 때까지 기다리는 것
- 예를 들어 파일에서 데이터를 읽기 위해 read()를 호출함. 데이터가 있었다면 read()가 바로 리턴되지만, 아직 수신한 데이터가 없으면 read()는 데이터가 도착할 때까지 계속 기다리게 됨(블록). 
- 한곳에서 입출력이 블록되면 프로그램 전체가 블록되기 때문에, 다수의 입출력 처리 시에는 블록이 있는 입출력 함수 사용을 주의해야 함. (멀티 스레드, 멀티프로세스는 크게 상관이 없음)

<br>

**블록모드**: 소켓의 기본 모드로 소켓에 시스템 콜 호출 시 처리가 완료될 때까지 프로세스가 멈춰있는 것을 말함.
- **블록될 수 있는 소켓 관련 시스템 콜**: listen(), connect(), accept(), recv(), send(), read(), write(), recvfrom(), sendto(), close()

한 프로세스(스레드) 내에서 이루어지는 다중 처리 방법에는 3가지 종류가 있음

### 1. 폴링(Polling)
처리해야 할 작업들을 순차적으로 돌아가면서 처리하는 방법
- 서버가 각 클라이언트의 요청을 순차적으로 처리함.
- 각 클라이언트가 고르게 요청하는 경우 적합
- 입출력 함수가 블록되지 않도록 non-block 모드로 설정해야 함. 

<br>

### 2. 셀렉팅(Selecting)
요청이 오면 해당 클라이언트의 입출력을 처리하는 방법. 
- 클라이언트의 요청이 불규칙적인 경우에 적합
- select() 함수를 이용하여 소켓을 비동기 모드로 변경해야 함.
- 비동기 모드: 소켓에서 I/O 변화가 발생하면 이를 알리는 모드 

소켓(파일)이 여러 개일 때, 어떤 놈한테서 데이터가 올지 미리 알 수가 없음. 그래서 select() 함수를 사용해서 소켓들을 감시하다가 어떠한 변화가 생기면 깨워달라고 지시하는 것임.
```C
int select(int nfds, fd_set *readfds, fd_set *writefds, fd_set *exceptfds, struct timeval *timeout)
```
- **fd_set**: 비트 배열(0 or 1), 감시하고 싶은 파일 식별자 번호에 해당하는 칸을 1로 바꿈
- **nfds**: 검사할 파일 디스크립터의 범위
- **readfds**: 누가 나한테 데이터를 보냈나 감시. 0번(stdout) 세팅: 사용자가 키보드를 쳤는지, 3번(소켓) 세팅: 클라이언트가 데이터를 보냈는지.
- **writefds**: 지금 데이터를 보낼 수 있는 상태인가를 감시. 
- - **exceptfds**: 예외 상태 변화를 감지
- **timeout**: select 함수가 대기하는 시간 설정 (NULL인 경우는 영구 대기)

<br>

### 3. 인터럽트
프로세스가 어떤 작업을 처리하는 중에 특정 이벤트가 발생하면 해당 이벤트를 처리하는 방식으로, 프로세스간 이벤트 전달에 시그널을 사용함.
