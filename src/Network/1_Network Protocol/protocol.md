# 0. 네트워크 프로토콜
**프로토콜**이란 데이터를 원활히 주고받을 수 있도록 정한 약속이다. 같은 통신 프로토콜을 지원하는 장비간에만 통신이 가능하다. <br>
표준 통신 프로토콜에는 HTTP, TCP/IP 등이 있다.

<br>

# 1. OSI 7 계층과 TCP/IP 프로토콜 구조
네트워크 모델은 이론적인 OSI 7계층과 실제 쓰이는 TCP/IP 4계층으로 나뉜다. 

### 1) 데이터 단위와 주소체계
1) **필드**: 헤더나 트레일러에 담긴 세부 정보 단위
2) **SDU(Service Data Unit)**: 상위 계층에서 온 순수한 데이터 알맹이
3) **PDU(Protocol Data Unit)**: SDU + 헤더/트레일러
   - 물리: 비트 전송
   - 데이터 링크: 프레임 전송
   - 네트워크: 패킷 전송
   - 전송: 데이터 송수신 및 오류/흐름 제어
4) **IP 주소**: 인터넷상 호스트 식별 (IPv4 32bit, IPv6 128bit).
5) **포트(Port)번호**
   - 포트(Port): 16비트 숫자로, 특정 소켓(접속점)을 구분. 같은 번호를 TCP/UDP가 써도 무방
   - 문자열 $\leftrightarrow$ 숫자 변환: 우리는 보통 "127.0.0.1"처럼 글자로 IP를 쓰지만, 컴퓨터는 32비트 숫자로 이해함. inet_addr() 같은 함수로 바꿔주는 과정이 필수

<br>

### 2-1) OSI 7 계층 구조
- **전송 계층**:
  - 네트워크 계층을 이용하여 종점 호스트 사이 데이터 송수신.
- **네트워크 계층**:
  - 패킷을 목적 Host까지 전달하는 교환 기능
  - 주소 분석, 경로 배정(라우팅)
- **데이터 링크 계층**:
  - 프레임(PDU)을 노드 사이에 신뢰성 있게 전송
- **물리 계층**:
  - 전송매체를 통하여 비트를 전송

<br>

### 2-2) TCP/IP 프로토콜 구조 
- **응용계층(L7)**: 택배 알맹이(데이터)를 준비하는 단계
- **전송 계층(L4)**: 어떤 방식으로 보낼지 정함. 동기(TCP)로 보낼지, UDP로 보낼지 결정.
- **네트워크 액세스 계층**:
  - 서브 네트워크에서 MAC 주소를 이용하여 프레임 단위로 실제 송수신하는 프로토콜 
  - MAC 주소: 네트워크 인터페이스(유무선 랜카드)에 할당된 48비트 크기의 H/W 고유주소
  - 이더넷, 와이파이 등
- **인터넷 계층**:
  - 인터넷 프로토콜: IP 주소에 명시된 목적지까지 패킷을 전달. 라우팅 수행
  - IP 주소: 인터넷에서 host를 식별하는 주소(IPv4 32bit, IPv6 128bit 등)
  - ICMP, ARP(IP 주소를 MAC 주소로), RARP(MAC → IP), IGMP	
  
<br>

### 3) Ehternet (이더넷)
우리가 흔히 쓰는 유선 랜(LAN)의 규칙

|Preamble|SFD|DA|SA|Type/Length|Data|FCS
|--------|---|---|--|---|---|---|
|7바이트|1바이트|6바이트|6바이트|2바이트|46~1500바이트|4바이트
- Ethernet 프레임 구조:
  - Preamble + SFD(Start Frame Delimiter): 비트 동기화
  - MAC 주소: 수신(Destination)과 발신(Source) 주소로 각각 6바이트씩 사용
  - Type/Length: 상위 프로토콜(IP, ARP 등) 종류나 길이를 나타냄
  - Data: 상위 계층에 전달될 데이터
  - FCS: 데이터 오류 검출을 위한 CRC 값

# 2. IP 프로토콜
서브네트워크를 이용하여 IP datagram을 호스트 간에 전달하는데, 비연결형이며 데이터의 손실이나 순서를 보장하지 않음 <br>

**라우팅**: IP 데이터 그램을 목적지 호스트까지 전달하기 위해 경로(라우터)를 정하는 것
![Image](/img/ip.png)

**IP 헤더 필드**:
- IHL: 헤더 크기로 최소 5 = 20 바이트
- TTL(Time To Live): 패킷이 네트워크를 무한 뺑뺑이 돌지 않게 막아주는 카운터
- Protocol: 상위 게층이 TCP인지 UDP인지 알려줌
  
**IP 주소 클래스**:
- Class D: 224~239 범위로 멀티캐스트용 주소

**MTU(Max Transmission Unit)**:
- 한 번에 보낼 수 있는 데이터 그램의 최대 크기로 이더넷은 1500 바이트

**ARP**:
- IP 주소를 통해 상대방의 물리 주소(MAC)를 알아내는 프로토콜
  
<br>

### 2-1) TCP 프로토콜

**특징:**
- 종점 호스트 간 연결형 전송 서비스
- 바이트 스트림: 데이터를 물줄기 처럼 보내며 수신 측은 데이터의 경계를 알 수 없음
- 양방향 스트림 제공
- 신뢰성 있는 데이터 전달
- 연결 설정 및 종료
   - 3-way handshake (연결): SYN(요청 X) → SYN(Y) + ACK(수락 X+1) → ACK(확인 Y+1)
   - 4-way handshake (종료): FIN(요청 X) → AKC(수락 X+1) ... FIN(요청 Z) → ACK(확인 Z+1). 즉, FIN과 ACK를 주고받으며 연결을 끊음. 
   - TIME-WAIT: 먼저 종료를 요청한 쪽이 마지막 ACK를 보낸 후 대기하는 상태.

<br>

### 2-2) UDP 프로토콜

**특징:**
- 확인 응답(ACK)이나 순서 보장이 없는 비연결형 서비스
- 데이터그램 단위로 주고 받으며 sendto 한 번에 recvfrom 한 번이 매칭됨.
- 헤더 크기: 8바이트
- 메시지 최대 크기 → 수신 버퍼 크기를 넘어가면 나머지 부분은 버려짐

# 3. 바이트 정렬
컴퓨터마다 숫자를 메모리에 저장하는 방식이 달라서 생기는 문제이다.

1) 호스트 바이트 순서: 컴퓨터가 내부 메모리에 정수를 저장하는 순서로 CPU 아키텍처에 따라 다름.
   - 참고: (80x86: little-endian | MC68000: big-endian | ARM: little-endian / big-endian)
   - Little-endian: 하위 바이트를 먼저 저장 (80x86, ARM 등).
   - Big-endian: 상위 바이트를 먼저 저장 (MC68000, ARM 등).
2) 네트워크 바이트 순서: port 번호나 IP주소와 같은 정보를 바이트 단위로 전송하는 순서. 무조건 **High-order(Big-endian)**로 전송.
3) 변환 함수:
   - htons() / ntohs() : host to network 바이트 변환. Unsigned short Integer(2바이트, Port 번호용).
   - htonl() / ntohl() : network to host 바이트 변환. Unsigned long Integer(4바이트, IP 주소용).

기타: IP 주소(4바이트)를 점진적 10진수(dotted decimal)로 바꾸는 함수들.
<br><br>

### 바이트 순서
   - Host Order: 내 컴퓨터 저장 방식 (Intel 계열은 보통 Little-endian).
   - Network Order: 전 세계 공통 약속 (Big-endian).
   - 변환 함수: htons, ntohs (2바이트/포트용), htonl, ntohl (4바이트/IP용).

서브 네트워크: 이더넷, 패킷, 교환망 등 실제 데이터를 실제로 전달해주는 네트워크
