머신 러닝을 진행하기 위해서 다음 4가지가 필요

**1. 데이터**
- 일반적으로 데이터가 많아질 수록 일이 더 쉬워짐. 더 많은 데이터가 있으면 더 강력한 모델을 학습 시킬 수 있음.

**2.모델**
- 데이터를 어떻게 변환할지

**3. 손실 함수**
- 우리가 얼마나 잘하고 있는 지를 측정하는 함수
- 예를 들어 실제 심장 박동은 60bpm인데 100bpm이라고 예측했다면 모델이 틀렸다는 걸 알려줘야 함<br><br>

**4. (최적화) 알고리즘**
- 예측이 틀렸을 때 모델에게 알려줄 방법이 필요함. 그래서 일반적으로 머신 러닝의 학습 부분은 이 손실 함수를 최소화하는 것으로 구성<br><br>
- 모델은 많은 파라미터를 갖는데, 이 파라미터들의 가장 좋은 값은 관찰된 데이터의 학습 데이터에 대한 손실(loss)를 최소화하는 것을 통해 배우는 것
	- **1) 학습 오류**: 학습 데이터에 대해 loss를 최소화 하면서 모델을 학습 시킨 데이터에 대한 오류 (모의고사를 잘 본다고 실제 시험에서 잘 본다는 보장은 없음). 학습 오류가 낮다는 것은 내가 준 데이터 안에서는 답을 잘 맞춘다는 뜻으로, 모델이 주어진 데이터에 대해 최적화되었다는 뜻일 뿐 그게 일반화된 성능인지는 아직 모름. <br>
	
	- **2) 테스트 오류**: 보지 않는 테스트 셋에 대한 오류. 보지 않는 데이터에 대한 일반화를 실패한 경우를 오버피팅(과적합)이라고 함. (모의고사는 잘 봤는데 실제 시험은 망침). 아무리 학습 데이터를 완전히 숙달했더라도 처음 보는 유형이 나오면 틀릴 수 있음 이때 틀리는 정도를 말함.
