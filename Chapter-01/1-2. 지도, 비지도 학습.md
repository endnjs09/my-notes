# 1. 지도 학습

**지도 학습**: 주어진 입력 데이터($X$)에 대한 타겟($y$)을 예측하는 문제를 푸는 것으로, 입력 $x$를 $f(𝜃)$로 매핑하는 $f(𝜃)$를 생성하는 것이 목표.

즉, 어떤 문제를 던져줘도 모델이 정답($y$)에 아주 가까운 예측 값을 뱉어낼 수 있도록 똑똑한 함수를 만들어 내는 것. (정답($y$)이 적혀있는 문제지($X$)를 모델에게 주고 모델이 정답을 맞히는 원리(𝜃)를 스스로 깨우치게 만드는 과정)

𝜃 에는 가중치($W$)와 편향($b$)가 있고 이것들을 조절 $𝜃 = {W, b}$
1) 가중치($W$): 입력 데이터가 결과에 얼마나 큰 영향을 주는지 결정하는 중요도
2) 편향($b$): 뉴런의 활성화 조건을 결정해주는 상수(매개변수)


**지도 학습 알고리즘**: 데이터 셋을 받아서 모델을 내놓는 함수

1) 재료(데이터 셋): 수만 개의 (문제 $X$, 정답 $y$) 쌍을 준비
2) 공장(알고리즘): 이 데이터를 지도 학습 알고리즘이라는 공장에 넣음
3) 제품(모델): 새로운 문제를 던져주면 답을 내뱉는 완성된 인공지능 모델

### 1-1. 지도 학습 종류 ###

**1. 회귀**:  '이 집이 얼마야?', '내일 주식 얼마야?', '생존 확률 얼마야?' 같은 '얼마나'(수치)를 맞히는 게임.
	- 원인($X$)과 결과($y$) 사이의 규칙을 찾아내서 새로운 $X$가 들어왔을 때 $y$가 얼마인지 숫자로 찍어주는 것
	- 1) 속성 벡터($X$): [평수, 방 개수, 역과의 거리] 같은 숫자 묶음 (데이터)
	- 2) 타겟($y$): 우리가 알고 싶은 결과 값. 위의 예시에서는 집 값과 같이 연속적인 숫자
	- 3) 예측값($\hat{y}$): 모델이 예상해본 결과 값 (진짜 $y$랑 비슷할수록 똑똑한 모델)


2. **멀티 레이블**: 회귀가 ‘얼마나 많이?’를 맞추는 거라면 분류는 ‘이게 뭐야?’라고 이름표를 붙이는 게임. 예를 들어 개와 고양이 분류 문제(이진 분류), 숫자 분류(다중 클래스 분류) 등


3. **태깅**: 이진, 다중 클래스 분류 문제 형태로 딱 떨어지지 않음. 예를 들어 '이 사진에 있는 동물은 개, 고양이, 닭 중 어느 것 입니까?', '이 사진에는 어떤 것들이 있습니까?' 등 멀티 레이블 분류 문제에서는 $X$가 포함될 확률이 얼마인지 개별적으로 계산하기 때문에 모든 확률의 합이 1을 넘을 수 있음.


4. 검색 & 태깅: '이게 뭐냐' 보다 '뭐가 먼저냐'. 즉, 검색 결과들 중에 어떤 것이 사용자에게 먼저 보여야 하냐를 선택하는 문제.
	- 분류에서는 A, B, C라는 결과가 나왔을 때 셋 다 어떠한 확률이 있다고만 하고 끝낸다면, 여기서는 순서가 A > B > C가 맞는지까지 따짐. 즉, 개별 데이터의 절대적인 값보다 데이터들 사이의 상대적인 우선순위를 결정하는 게 목표


5. 추천 시스템: 검색&랭킹은 모든 사람에게 비슷한 정답을 순서대로 보여주는 거라면, 추천 시스템은 나 자신이라는 필터를 한 번 더 거침. 유튜브의 영상 추천 알고리즘 등

# 2. 비지도 학습

**비지도 학습**에서는 패턴을 가능한 빨리 인지해서 행동을 모방. 누군가 데이터 뭉치를 주고 알아서 특징 좀 뽑아와 봐 같은 상황임. 정답(y)가 없기 때문에 모델 스스로 데이터 안의 숨겨진 구조나 공통된 특징을 찾아야 함.

### 2.1 비지도 학습 종류
1. **클러스터링**: 비슷한 놈들끼리 모으기
	- 데이터를 관찰해서 스스로 그룹을 나누는 작업
	- 과목 이름 데이터를 넣었을 때 이건 과학, 이건 수학 등 유형별로 묶어주는 것
    
2. **주성분 분석(PCA)**: 데이터의 차원을 축소 하는 기술, 정보의 손실을 최소화 하면서 가장 중요한 특징만 골라냄.
	- 과목 이름, 전공, 출석, 중간 점수 등 정보가 있을 때 결국 성적을 결정짓는 가장 핵심적인 2가지 요인은 무엇인가를 찾아내어 데이터를 단순화할 수 있음

3. **표현학습**: 개념 사이의 학적 관계. 데이터를 숫자로 바꾸었을 때 그 숫자들 사이에 논리적인 연산이 가능하게 만드는 것.
	- 단어들을 좌표 공간에 배치했더니 국가와 수도라는 관계성이 일정한 방향과 거리로 나타나는 현상. 딥러닝이 글자를 이해하는 가장 기초적인 방식

4. **인과관계**(방향성 그래프 모델): ‘A 때문에 B가 일어났다’를 분석하는 것
	- 주택 가격이 오른 게 위치 때문인지 물가 상승 때문인지 그 뿌리 원인을 추적


5. **GAN(Generate Adversarial Network)**: 실제 데이터와 가짜 데이터가 같은지 점검
	- GAN을 이용해 실제 학생 데이터와 통계적으로 거의 똑같은 가짜 데이터 8천개를 생성하여 1만 개로 학습 시킬 수 있음.
