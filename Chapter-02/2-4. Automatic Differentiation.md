2-3에선 미분이란 게 이런 거다를 보여주는 이론이었다면, 여기서는 그 미분을 Pytorch가 어떻게 해주는가에 대한 것을 다룬다. 딥러닝의 핵심인 역전파가 실제로 어떻게 돌아가는지 정리해보겠다.

# 1. 계산 그래프와 역전파
계산 그래프란 계산 과정을 그래프로 나타낸 것이다. 우리가 코드를 짜면 Pytorch는 내부적으로 '누가 누구랑 곱해서 어떠한 결과가 되었냐'를 기록한다.

```python
x = torch.arange(4.0, requires_grad=True)   # requires_grad=True: 연산 과정을 기록할 것인가
y = 2 * torch.dot(x, x)                     # 결과값은 28이 나옴          

y.backward()    # 연쇄 법칙을 가장 뒤에서부터 거꾸로 타고 올라감
x.grad          # 계산된 최종 미분값이 .grad 속성에 도착 
```
```python
tensor([ 0.,  4.,  8., 12.])
```

backward()가 핵심이다. 오차(Loss)와 같은 마지막 결과물에서 backward()를 실행하면 아까 기록해둔 그래프를 타고 거꾸로 올라가면서 모든 변수의 미분값을 계산한다. 이때 연쇄 법칙을 가장 뒤에서부터 거꾸로 적용해 나가는 것이다.
<br><br><br>

# 2. 비-스칼라(Non-Scalar) 변수의 미분
결과 값이 숫자 하나(Scalar)가 아니라 여러개(Vector or Matrix)일 때는 어떻게 미분할까.

### 1) Scalar를 Vector로 미분 (Gradient)
스칼라 함수 $f$를 $n$차원 벡터 $\mathbf{x} = [x_1, x_2, \dots, x_n]^T$로 미분하는 경우이다. 결과는 각 원소에 대한 편미분을 모아놓은 그래디언트(Gradient) 벡터가 된다.
$$\nabla f(\mathbf{x}) = \frac{\partial f}{\partial \mathbf{x}} = \begin{bmatrix} \frac{\partial f}{\partial x_1} \\ \frac{\partial f}{\partial x_2} \\ \vdots \\ \frac{\partial f}{\partial x_n} \end{bmatrix}$$
함수 $f$가 각 방향으로 얼마나 빠르게 변하는지를 나타내는 방향벡터를 의미한다.
<br><br>
### 2) Vector를 Vector로 미분 (Jacobian)
스칼라 $f$를 $m \times n$ 행렬 $\mathbf{X}$로 미분하면, 결과 역시 $m \times n$ 크기의 행렬이 된다. 야코비안(Jacobian) 행렬이라고 한다.
$$\frac{\partial f}{\partial \mathbf{X}} = \begin{bmatrix} \frac{\partial f}{\partial X_{11}} & \dots & \frac{\partial f}{\partial X_{1n}} \\ \vdots & \ddots & \vdots \\ \frac{\partial f}{\partial X_{m1}} & \dots & \frac{\partial f}{\partial X_{mn}} \end{bmatrix}$$
예를 들어 손실 함수를 가중치 행렬로 미분하여 가중치를 업데이트할 때 사용한다.
<br><br>

다만 위 방식은 비효율적일 수 있다. 입력($x$)이 100만개이고 출력($y$)도 100만개라면 야코비안 행렬은 100만 x 100만 = 1조개의 칸을 가진 엄청나게 큰 표가 될 것이다. 하지만 우리의 최종 목적은 오차(Loss)라는 숫자 하나를 줄이는 것이기 때문에 Pytorch에서는 벡터-야코비안 곱(Vector-Jacobian)이라는 방식을 사용한다. 
```python
y.sum().backward()
```
전체 표를 다 그리지 말고, 최종 목적지인 오차(Loss)에서 거꾸로 올라와 보자는 전략이다. backward()는 결과값이 숫자 하나(Scalar)여야 실행할 수 있도록 설계되어 있는데, 결과($y$)가 벡터라면 당연히 실행되지 않을 것이다. 이때 y.sum()으로 여러 개의 결과($y$)를 다 더해서 하나의 숫자로 합친다. 숫자가 하나가 되었으니 역전파를 할 수 있게 되는 것이다. 
<br><br><br>

# 4. Detaching
Detach는 쉽게 말해 '신분세탁'이다. 우리가 연산을 할 때 Pytorch는 뒤에서 로그(계산 그래프)를 남기는데 detach()는 그 로그를 중간에서 잘라버리는 역할을 한다. 
```python
x.grad.zero_()
y = x * x
u = y.detach()  # u는 y와 값은 똑같지만 x로부터 왔다는 기록이 삭제된 상수로 취급됨
z = u * x       # 그래서 z를 미분할 때 u를 그냥 상수로 취급함

z.sum().backward()
x.grad == u
```
```python
tensor([True, True, True, True])
```
detach()를 쓰면 현재까지의 계산 그래프에서 이 변수를 떼어낸다. 이후 연산은 기록되지 않기 때문에 미분할 때 상수로 취급된다.
